The artifacts are the only output for data.

So we need a frontend for 

1. adding some data to the artifacts, such as notes, such as (don't do this!) - tags consumed by export 
(such as adding an interception point, guard, OOD detection)
2. deleting trees of artifacts (when a whole bunch of stuff is wrong)
3. rule-based pruning of artifacts
4. AI-based analysis of artifacts
5. Exporting the artifacts for consumption by a model to be tuned (with options)
6. Exporting and just going ahead and fine tuning a model on OpenAI, HF, Claude - why not?

So in particular, this is where the platform really starts to take shape as a generalizable piece of software. We can
then do any back-office thing here, extracting any process with any set of skills, etc.

This allows for further curation of data.

In particular, this is a key part of the process of extracting into fine-tuned models.
